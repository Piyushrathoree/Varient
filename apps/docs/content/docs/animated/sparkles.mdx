## Installation

Copy the component from `packages/ui/src/components/animated/sparkles/` into your project.

```bash
bun add motion clsx tailwind-merge
```

## Usage

Wrap content to decorate it, or use `className` alone for a standalone sparkle
layer:

```tsx
import { Sparkles } from '@/components/animated/sparkles';

export function Example() {
  return (
    <Sparkles className="rounded-xl border border-border px-8 py-12">
      <div className="flex flex-col items-center gap-3 text-center">
        <h2 className="text-2xl font-semibold">
          Make it <span className="text-brand">Sparkling</span>
        </h2>
        <button type="button">Shine on</button>
      </div>
    </Sparkles>
  );
}
```

## Accessibility

- Sparkle particles are decorative — the overlay uses `aria-hidden="true"`
  and `pointer-events-none`, so they never intercept clicks or focus.
- Wrapped content stays fully interactive in a `relative z-10` layer.
- Under `prefers-reduced-motion`, no animated sparkles spawn. When `children`
  are present, only the content renders. Standalone layers show a few static
  faint stars instead of the twinkle loop.
- Spawn positions are seeded via a deterministic PRNG, so server and client
  markup match on first paint (no hydration mismatch).
- The spawn interval pauses automatically while the component is scrolled
  offscreen, and is cleared entirely on unmount along with the particle pool.
