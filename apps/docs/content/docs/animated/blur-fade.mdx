## Installation

Copy the component from `packages/ui/src/components/animated/blur-fade/` into your project.

```bash
bun add motion clsx tailwind-merge
```

## Usage

Wrap any content to reveal it with a blur-fade entrance. Stagger siblings with
incremental `delay` values:

```tsx
import { BlurFade } from '@/components/animated/blur-fade';

export function Example() {
  return (
    <div className="grid gap-4">
      <BlurFade delay={0}>
        <div className="rounded-xl border border-border bg-card p-6">First</div>
      </BlurFade>
      <BlurFade delay={0.1}>
        <div className="rounded-xl border border-border bg-card p-6">Second</div>
      </BlurFade>
    </div>
  );
}
```

## Performance

`filter: blur()` is a compositor-expensive property — noticeably heavier than
animating `transform`/`opacity` alone. It's fine for a handful of staggered
reveals, but stacking many `BlurFade` instances in one dense grid (dozens of
cards animating blur simultaneously) can visibly tax lower-end devices.
Prefer smaller `blur` values, stagger delays generously, or pass `blur="0px"`
for large grids where only the slide/fade matters.

## Accessibility

- Content is always present in the DOM — only opacity, blur, and transform animate.
  Screen readers can access children regardless of animation state.
- Under `prefers-reduced-motion`, children render immediately with no blur or
  slide transform.
- When `inView` is true (default), the animation runs once via `useInView` with
  `once: true` and a `-80px` margin — refresh the page to replay. Set
  `inView={false}` to reveal on mount instead of scroll.
