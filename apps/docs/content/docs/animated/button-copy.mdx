## Installation

Copy the component from `packages/ui/src/components/animated/button-copy/` into your project.

```bash
bun add motion clsx tailwind-merge
```

## Usage

```tsx
import { ButtonCopy } from '@/components/animated/button-copy';

export function Example() {
  return <ButtonCopy content="npx varient add button-copy" />;
}
```

Pass `onCopy` instead of `content` for custom copy logic (e.g. copying
rendered code rather than a raw string) — `onCopy` takes priority when both
are set:

```tsx
<ButtonCopy onCopy={async () => copyRenderedSnippet()} />
```

Set `display="label"` to widen into a pill with a text label next to the
icon — the button's width animates smoothly as the label changes length:

```tsx
<ButtonCopy display="label" content="npx varient add button-copy" />
```

## Accessibility

- In icon mode, the accessible name lives in a visually-hidden
  `aria-live="polite"` span, so "Copy" → "Copying" → "Copied" is announced to
  screen readers as the state changes, without depending on assistive tech
  re-reading a changing `aria-label`. In label mode, the visible text is the
  accessible name and the button itself is the `aria-live="polite"` region.
- Interaction is disabled while copying or while showing the copied
  confirmation, preventing duplicate clipboard writes mid-animation.
- The icon morph (copy → spinner → check) is purely decorative and marked
  `aria-hidden`; the live-region text carries the actual state.
- The loading spinner is a genuine round SVG ring (a partial circular arc
  with round line caps) that spins continuously; the rotation is skipped
  under `prefers-reduced-motion` but the ring itself still renders, keeping
  the state visually distinct.
- The "copied" checkmark's `text-emerald-500` accent is a deliberate,
  scoped exception to the semantic token palette — it matches the app's own
  copy-button confirmation color.
