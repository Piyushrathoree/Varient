## Installation

Copy the component from `packages/ui/src/components/animated/cursor-spotlight/` into your project.

```bash
bun add motion clsx tailwind-merge
```

## Usage

`CursorSpotlight` renders `children` exactly once, at full brightness and
fully interactive. A decorative scrim overlay dims and desaturates the
surrounding area, punched through by a circular hole that follows the
pointer via a radial CSS mask:

```tsx
import { CursorSpotlight } from '@/components/animated/cursor-spotlight';

export function Example() {
  return (
    <CursorSpotlight className="h-56 rounded-xl border border-border bg-background">
      <div className="grid grid-cols-3 gap-2 p-4">
        {['Motion', 'Tokens', 'Radix'].map((word) => (
          <span key={word} className="rounded-lg border border-border bg-card px-3 py-2 text-sm">
            {word}
          </span>
        ))}
      </div>
    </CursorSpotlight>
  );
}
```

## Accessibility

- `children` render exactly once in the DOM, at full opacity, with no
  `aria-hidden` or `pointer-events-none` applied — every interactive element
  inside stays keyboard-reachable and works the same as if it weren't
  wrapped in `CursorSpotlight`.
- The dim/reveal effect is achieved entirely by a decorative scrim overlay
  (`aria-hidden="true"`, `pointer-events-none`) layered on top of the real
  content; it never duplicates or intercepts it.
- Requires a fine, hover-capable pointer (`(hover: hover) and (pointer:
  fine)`, checked via `useFinePointer`) — touch devices render children
  plainly with no scrim, since there's no cursor to drive the reveal.
- Under `prefers-reduced-motion`, children render at full brightness with no
  mask and no pointer tracking listeners attached.
