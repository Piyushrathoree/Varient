## Installation

Copy the component from `packages/ui/src/components/animated/carousel/` into your project.

```bash
bun add motion clsx tailwind-merge
```

## Usage

```tsx
import { Carousel } from '@/components/animated/carousel';

export function Example() {
  return (
    <Carousel align="center" gap={16} className="w-full max-w-md">
      <Carousel.Item className="w-full">Slide one</Carousel.Item>
      <Carousel.Item className="w-full">Slide two</Carousel.Item>
      <Carousel.Item className="w-full">Slide three</Carousel.Item>
      <Carousel.Previous />
      <Carousel.Next />
      <Carousel.Dots />
    </Carousel>
  );
}
```

Slide width is controlled entirely by the `className` on each `Carousel.Item` — use `w-full` for
one-at-a-time, or a percentage (`w-[70%] sm:w-[55%]`) for a centered "peek" layout where neighboring
slides are partially visible. `Carousel.Previous`, `Carousel.Next`, and `Carousel.Dots` can be placed
anywhere inside `<Carousel>` — they're rendered outside the draggable track automatically.

## Accessibility

- The viewport is `role="region"` `aria-roledescription="carousel"`, focusable, with `ArrowLeft`/`ArrowRight` navigating slides.
- Each `Carousel.Item` is `role="group"` `aria-roledescription="slide"` with an `aria-label` announcing its position (`"2 of 5"`), computed automatically from render order.
- `Carousel.Previous` / `Carousel.Next` carry `aria-label` and are real `<button disabled>` elements at the bounds (non-looping).
- `Carousel.Dots` buttons use `aria-label="Go to slide N"` and `aria-current` on the active dot.
- Under `prefers-reduced-motion`, drag still works but the release/settle snap is instant — no spring overshoot.
