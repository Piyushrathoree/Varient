## Installation

Copy the component from `packages/ui/src/components/animated/expandable-card/` into your project.

```bash
bun add motion clsx tailwind-merge
```

## Usage

```tsx
import { ExpandableCard } from '@/components/animated/expandable-card';

export function Example() {
  return (
    <ExpandableCard
      title="Designing for orientation"
      description="How wayfinding cues keep users grounded in deep navigation."
      media={<img src="/cover.jpg" alt="" />}
      footer={<span className="text-xs text-muted-foreground">5 min read</span>}
    >
      The full article body, revealed once the card has expanded into a
      centered panel.
    </ExpandableCard>
  );
}
```

The collapsed card shows `media` (optional), `title`, and `description`.
Clicking it — or focusing it and pressing Enter/Space — morphs it into a
centered dialog that also shows `footer` and the `children` passed as the
expanded body. `media` and `title` share a `layoutId` between both states
(scoped per instance via `useId`, so multiple cards on the same page never
collide), so the shape animates continuously rather than cross-fading two
unrelated elements.

Omit `media` for a compact, text-only card:

```tsx
<ExpandableCard title="Release notes — v2.4" description="What shipped this week.">
  Full changelog contents…
</ExpandableCard>
```

## Accessibility

- The collapsed card is `role="button"` with `tabIndex={0}`, `aria-haspopup="dialog"`,
  and `aria-expanded` — reachable and activatable by keyboard (Enter or Space)
  as well as click, with a visible `focus-visible` ring.
- The expanded panel is `role="dialog"` `aria-modal="true"`, labelled by its
  title via `aria-labelledby`.
- Opening focuses the visible close button; closing (via Escape, the overlay,
  or the close button) restores focus to the collapsed card.
- Escape closes the panel; body scroll is locked while it's open.
- While expanded, the collapsed card is kept mounted (not unmounted) but made
  `invisible` and `aria-hidden` with `tabIndex={-1}` — this keeps the grid
  from reflowing and gives motion a stable origin to animate back to, without
  leaving a second focusable/announced copy in the page.
- Under `prefers-reduced-motion`, the shared `layoutId`s are omitted entirely
  — the panel plain-fades open/closed in place instead of morphing.
