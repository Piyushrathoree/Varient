## Installation

Copy the component from `packages/ui/src/components/sections/cta/` into your project.

```bash
bun add motion clsx tailwind-merge
```

## Usage

```tsx
import { Cta } from '@/components/sections/cta';

export function PricingPage() {
  return (
    <Cta
      eyebrow="Varient Pro"
      title="Ready to get started"
      description="Copy components directly into your project. No runtime dependency — you own every line."
      primaryCta={{ label: 'Browse components', href: '/components' }}
      secondaryCta={{ label: 'View on GitHub', href: 'https://github.com' }}
      secondaryCtaEmphasis="ghost"
      socialProof={{
        avatars: [{ src: '/avatars/1.png' }, { src: '/avatars/2.png' }],
        text: 'Trusted by 2,400+ engineering teams',
      }}
    />
  );
}
```

## Variants

Three surface styles via the `variant` prop:

```tsx
<Cta variant="minimal" title="Varient 2.0 is here" primaryCta={{ label: 'Read the release notes', href: '/changelog' }} />
```

- **`default`** — `bg-card` with a hairline border and soft shadow. Neutral, works anywhere on the page.
- **`brand`** — same card fill with a subtle brand gradient border rim. Use sparingly for high-intent conversion moments.
- **`minimal`** — borderless and transparent, no card fill or shadow. Use on pages that already have enough surfaces (e.g. stacked directly on the page background between other sections).

## Secondary CTA emphasis

The secondary action's visual weight is controlled independently via `secondaryCtaEmphasis`:

```tsx
<Cta
  primaryCta={{ label: 'Start building', href: '/components' }}
  secondaryCta={{ label: 'Talk to sales', href: '/contact' }}
  secondaryCtaEmphasis="link"
/>
```

- **`outline`** (default) — bordered button, equal footing with the primary action.
- **`ghost`** — borderless, fills on hover. Quieter than outline.
- **`link`** — text-only underline. Lowest visual weight, good when the secondary action is truly optional.

## Social proof

Pass `socialProof` to render an overlapping avatar stack with a trust line beneath the actions:

```tsx
<Cta
  socialProof={{
    avatars: [{ src: '/avatars/1.png', alt: 'Amara O.' }, { src: '/avatars/2.png' }],
    text: 'Trusted by 2,400+ engineering teams',
  }}
/>
```

`avatars` is optional — omit it to render the trust line alone. Only the first 5 avatars are rendered.

## Accessibility

The root is a `<section>` labelled by the h2 via `aria-labelledby`. Buttons are native anchors inside `Button asChild`. Avatars in the social-proof stack are decorative (`aria-hidden`) since the trust line already conveys the same information as text.

| Key | Action |
| --- | --- |
| `Tab` / `Shift + Tab` | Moves focus to the CTA link(s). |
| `Enter` | Activates the focused link. |

The scroll-reveal animation is gated behind `prefers-reduced-motion` — reduced-motion users see the banner immediately at full opacity.
