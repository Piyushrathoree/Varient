## Installation

Copy the component from `packages/ui/src/components/sections/changelog/` into your project.

```bash
bun add motion clsx tailwind-merge
```

## Usage

```tsx
import { Changelog } from '@/components/sections/changelog';

export function ReleasesPage() {
  return (
    <Changelog
      eyebrow="Changelog"
      title="What shipped recently"
      entries={[
        {
          version: '1.2.0',
          date: 'Jul 1, 2026',
          title: 'Sections layer ships',
          changes: [
            { type: 'added', text: 'Integration grid and 404 page sections.' },
            { type: 'improved', text: 'Card hover states with softer shadows.' },
          ],
        },
      ]}
    />
  );
}
```

## Accessibility

The section uses a semantic `<section>` with `aria-labelledby` pointing at the heading.
Each release renders as an `<article>` with a version row, optional `<h3>` title, and a semantic `<ul>` of changes.
Version dates use a `<time>` element for machine-readable dating when you pass an ISO-friendly `date` string.
Change-type badges are visible text labels — not icon-only.

Scroll-reveal animations are gated behind `prefers-reduced-motion` — reduced-motion users see content immediately without stagger or fade.
