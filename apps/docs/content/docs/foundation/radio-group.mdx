## Installation

Copy the component from `packages/ui/src/components/foundation/radio-group/` into your project.

```bash
bun add @radix-ui/react-radio-group motion clsx tailwind-merge
```

## Usage

```tsx
import { useState } from 'react';
import { RadioGroup } from '@/components/foundation/radio-group';

export function Example() {
  const [plan, setPlan] = useState('pro');

  return (
    <RadioGroup value={plan} onChange={setPlan}>
      <RadioGroup.Item value="free" label="Free" />
      <RadioGroup.Item value="pro" label="Pro" />
      <RadioGroup.Item value="team" label="Team" />
    </RadioGroup>
  );
}
```

`RadioGroup.Item` skips its built-in `label` (leave the prop out) when you need a richer click target — pass your own `id` and wrap a `<label htmlFor>` around whatever markup you want.

For the common "selectable tile" case, use `variant="card"` instead — it wraps the dot and a two-line `label`/`description` in a soft-cornered, fully clickable tile for you (this is what the plan-picker above uses):

```tsx
<RadioGroup value={plan} onChange={setPlan}>
  <RadioGroup.Item
    variant="card"
    value="pro"
    label="Pro"
    description="$19/mo — full animated layer, priority support."
  />
</RadioGroup>
```

RadioGroup is fully controlled — there is no internal selected-value state, so both `value` and `onChange` are required even when the group is disabled.

## Accessibility

RadioGroup renders Radix's WAI-ARIA radio group pattern: the root gets `role="radiogroup"` and each item gets `role="radio"` with `aria-checked`. Arrow Right/Down and Arrow Left/Up move focus between items and immediately select the newly-focused one (roving tabindex, automatic activation) — the axis follows `orientation`. When `label` is passed on an item, it renders as a `<label>` tied to that item through a `useId()`-generated (or caller-supplied) `id`/`htmlFor` pair. The selection dot's scale-in/out animation drops to a duration of `0` when `prefers-reduced-motion` is set.
