## Installation

Copy the component from `packages/ui/src/components/foundation/drawer/` into your project.

```bash
bun add @radix-ui/react-dialog motion clsx tailwind-merge
```

## Usage

```tsx
import { Drawer, Button } from '@/components/foundation/drawer';

export function Example() {
  return (
    <Drawer>
      <Drawer.Trigger asChild>
        <Button variant="outline">Open drawer</Button>
      </Drawer.Trigger>

      <Drawer.Content side="right" size="md">
        <Drawer.Header>
          <Drawer.Title>Filters</Drawer.Title>
          <Drawer.Description>Refine the list with the options below.</Drawer.Description>
        </Drawer.Header>

        {/* panel content */}
      </Drawer.Content>
    </Drawer>
  );
}
```

For controlled usage, pass `isOpen` and `onOpenChange` to `Drawer` instead of relying on `Drawer.Trigger` — useful when a save action or async flow should close the drawer programmatically.

```tsx
const [isOpen, setIsOpen] = useState(false);

<Drawer isOpen={isOpen} onOpenChange={setIsOpen}>
  <Drawer.Trigger asChild>
    <Button variant="outline">Settings</Button>
  </Drawer.Trigger>
  <Drawer.Content side="left">{/* ... */}</Drawer.Content>
</Drawer>;
```

## Variants

`Drawer.Content` accepts `side` and `size` props:

- **`side`** — `'right'` (default), `'left'`, `'top'`, or `'bottom'`. Controls which edge the panel slides in from.
- **`size`** — `'sm'`, `'md'` (default), `'lg'`, or `'full'`. Sets width for left/right drawers or height for top/bottom drawers.

Set `showCloseButton={false}` on `Drawer.Content` to hide the built-in X and supply your own via `Drawer.Close`.

## Accessibility

- Built on `@radix-ui/react-dialog` for focus trap, portal rendering, and dismiss handling.
- **Esc** closes the drawer; clicking the overlay closes it too (unless `modal={false}` changes outside-interaction behavior).
- Focus moves into the panel on open and returns to the trigger on close — Tab is trapped inside the panel while it's open.
- `Drawer.Title` renders Radix's `Title` primitive, which wires `aria-labelledby` on the panel automatically. `Drawer.Description` does the same for `aria-describedby`. Always include a `Drawer.Title` — if you want to hide it visually, wrap it with a visually-hidden utility rather than omitting it.
- All entrance/exit motion respects `prefers-reduced-motion` — reduced-motion users get an instant, un-animated state change instead of the slide/fade transition.
