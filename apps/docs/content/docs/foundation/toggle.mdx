## Installation

Copy the component from `packages/ui/src/components/foundation/toggle/` into your project.

```bash
bun add @radix-ui/react-toggle motion clsx tailwind-merge
```

## Usage

```tsx
import { Toggle } from '@/components/foundation/toggle';

export function Example() {
  return (
    <Toggle defaultPressed aria-label="Bold">
      Bold
    </Toggle>
  );
}
```

## Variants

Two visual styles, switched with the `variant` prop:

```tsx
<Toggle variant="outline" aria-label="Italic">
  Italic
</Toggle>
```

- **`default`** — a ghost button that fills with `bg-muted` when pressed. Good for toolbar icon toggles in a low-chrome row.
- **`outline`** — adds a `border-input` frame and a soft shadow. Reads well when the toggle sits on its own or beside other bordered controls.

Off state is transparent with a `hover:bg-muted` fill; on state is `bg-muted text-foreground` via Radix's `data-[state=on]`. Every press scales down to `0.97` before any async work — gated behind `prefers-reduced-motion` for users who need it.

## Accessibility

Built on `@radix-ui/react-toggle` for correct `aria-pressed` semantics and keyboard behavior.

| Key           | Action                                             |
| ------------- | --------------------------------------------------- |
| `Enter` / `Space` | Toggles the pressed state.                      |
| `Tab` / `Shift + Tab` | Moves focus to the next / previous focusable element. |

The root renders a `<button>` with `aria-pressed` reflecting the current state. A visible focus ring (`ring-ring`) appears on keyboard focus. The press scale animation is skipped when `prefers-reduced-motion` is set.
