## Installation

Copy the component from `packages/ui/src/components/foundation/dropdown-menu/` into your project.

```bash
bun add @radix-ui/react-dropdown-menu motion clsx tailwind-merge
```

## Usage

```tsx
import { LogOut, Settings, User } from 'lucide-react';
import { Button } from '@/components/foundation/button';
import { DropdownMenu } from '@/components/foundation/dropdown-menu';

export function Example() {
  return (
    <DropdownMenu>
      <DropdownMenu.Trigger asChild>
        <Button variant="outline">Account</Button>
      </DropdownMenu.Trigger>
      <DropdownMenu.Content>
        <DropdownMenu.Item icon={<User className="size-4" />}>
          Profile
          <DropdownMenu.Shortcut>⇧⌘P</DropdownMenu.Shortcut>
        </DropdownMenu.Item>
        <DropdownMenu.Item icon={<Settings className="size-4" />}>
          Settings
          <DropdownMenu.Shortcut>⌘,</DropdownMenu.Shortcut>
        </DropdownMenu.Item>
        <DropdownMenu.Separator />
        <DropdownMenu.Item variant="destructive" icon={<LogOut className="size-4" />}>
          Log out
        </DropdownMenu.Item>
      </DropdownMenu.Content>
    </DropdownMenu>
  );
}
```

Uncontrolled by default — the menu tracks its own open state. Pass `open` together with `onOpenChange` to control it externally.

## Accessibility

DropdownMenu follows the WAI-ARIA menu pattern via Radix:

- The trigger gets `aria-haspopup`, `aria-expanded`, and `aria-controls` pointing at the menu.
- Arrow Up / Arrow Down move the highlighted item; Home / End jump to the first / last item.
- Typeahead — typing jumps to the next item whose label starts with those characters.
- Escape closes the menu and returns focus to the trigger; clicking outside does the same.
- `modal` (default `true`) traps focus inside the open menu and blocks pointer events on the rest of the page.
- The scale + fade transition is skipped in favor of an instant swap when `prefers-reduced-motion` is set.
