## Installation

Copy the component from `packages/ui/src/components/foundation/accordion/` into your project.

```bash
bun add @radix-ui/react-accordion motion clsx tailwind-merge
```

## Usage

```tsx
import { Accordion } from '@/components/foundation/accordion';

export function Example() {
  return (
    <Accordion defaultValue="item-1">
      <Accordion.Item value="item-1">
        <Accordion.Trigger>What is Varient?</Accordion.Trigger>
        <Accordion.Content>
          A copy-paste component library, animated and accessible by default.
        </Accordion.Content>
      </Accordion.Item>

      <Accordion.Item value="item-2">
        <Accordion.Trigger>Can more than one item be open?</Accordion.Trigger>
        <Accordion.Content>
          Pass <code>isMultiple</code> to the root to allow several items open at once.
        </Accordion.Content>
      </Accordion.Item>
    </Accordion>
  );
}
```

## Variants

Three visual styles, switched with the `variant` prop — the compound API
(`Accordion.Item` / `Accordion.Trigger` / `Accordion.Content`) is identical
across all of them.

```tsx
<Accordion variant="separated" defaultValue="item-1">
  {/* ...items */}
</Accordion>
```

- **`default`** — a single bordered card with hairline dividers between rows. Good default for a settings page or sidebar.
- **`separated`** — every item is its own rounded card with a gap between them, lifting slightly when open. Reads well for FAQ sections that need visual separation.
- **`ghost`** — no frame at all; the active row gets a soft `bg-muted` fill instead of a border. Good for dense, low-chrome contexts.

## Accessibility

Built on `@radix-ui/react-accordion` for correct roles and keyboard behavior.

| Key           | Action                                             |
| ------------- | --------------------------------------------------- |
| `Enter` / `Space` | Toggles the focused item open or closed.          |
| `Tab` / `Shift + Tab` | Moves focus to the next / previous focusable element. |
| `Arrow Down`  | Moves focus to the next trigger.                    |
| `Arrow Up`    | Moves focus to the previous trigger.                |
| `Home`        | Moves focus to the first trigger.                   |
| `End`         | Moves focus to the last trigger.                    |

Each panel is a `role="region"` labelled by its trigger, and every open/close
transition is gated behind `prefers-reduced-motion` — reduced-motion users get
an instant toggle instead of the spring animation.
