`Tooltip` is intentionally a single component driven by a `content` prop, not a
compound API like `Popover` or `Dialog` — there's no `Tooltip.Trigger` /
`Tooltip.Content` to assemble yourself. The only other piece is the optional
`Tooltip.Provider` wrapper for sharing delay timing across a group. Reach for
`Popover` instead when you need richer, interactive, or focusable content.

## Installation

Copy the component from `packages/ui/src/components/foundation/tooltip/` into your project.

```bash
bun add @radix-ui/react-tooltip motion clsx tailwind-merge
```

## Usage

```tsx
import { Tooltip } from '@/components/foundation/tooltip';

export function Example() {
  return (
    <Tooltip content="Save changes" side="top">
      <button>Save</button>
    </Tooltip>
  );
}
```

The trigger is whatever single element you pass as `children` — it's merged onto the
element via Radix's `asChild`, so it must accept a `ref` (any Varient component or a
plain DOM element works).

Wrap a group in `Tooltip.Provider` to share `delayDuration` (and the hover skip-delay
window) across every tooltip beneath it — each `Tooltip` still works fully on its own
without one:

```tsx
import { Tooltip } from '@/components/foundation/tooltip';

export function Toolbar() {
  return (
    <Tooltip.Provider delayDuration={300}>
      <Tooltip content="Bold">
        <button>B</button>
      </Tooltip>
      <Tooltip content="Italic">
        <button>I</button>
      </Tooltip>
    </Tooltip.Provider>
  );
}
```

## Colors

Set `color` to map the bubble (and its arrow) to a semantic token. `default` is the
neutral inverse surface used above; the rest borrow status/brand tokens for
callouts inside toolbars, forms, and confirmation flows.

```tsx
<Tooltip content="Recommended" color="brand">
  <button>Upgrade</button>
</Tooltip>
<Tooltip content="Synced" color="success">
  <button>Save</button>
</Tooltip>
<Tooltip content="Needs review" color="warning">
  <button>Submit</button>
</Tooltip>
<Tooltip content="Cannot be undone" color="destructive">
  <button>Delete</button>
</Tooltip>
<Tooltip content="Matches the popover surface" color="surface">
  <button>Info</button>
</Tooltip>
```

## Animation variants

Set `variant` to change how the tooltip enters. `scale` is the default — the
original scale + fade + subtle directional pull. `slide` swaps that for a more
pronounced directional slide on a snappier spring. `fade` drops movement and
scale entirely for a soft opacity-only cross-fade.

```tsx
<Tooltip content="Slides in from the side" variant="slide">
  <button>Slide</button>
</Tooltip>
<Tooltip content="Just a soft cross-fade" variant="fade">
  <button>Fade</button>
</Tooltip>
```

## Accessibility

- Opens on both **hover** and **keyboard focus** of the trigger, and closes on blur or
  pointer-leave — never mouse-only.
- Pressing **Esc** dismisses an open tooltip.
- Radix wires `aria-describedby` from the trigger to the tooltip's `role="tooltip"`
  content automatically — icon-only triggers still need their own `aria-label` for an
  accessible name, since a description doesn't replace one.
- All motion is gated behind `useReducedMotion()` — reduced-motion users get an instant
  cross-fade regardless of `variant`.
