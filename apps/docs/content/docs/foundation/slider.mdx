## Installation

Copy the component from `packages/ui/src/components/foundation/slider/` into your project.

```bash
bun add @radix-ui/react-slider motion clsx tailwind-merge
```

## Usage

```tsx
import { useState } from 'react';
import { Slider } from '@/components/foundation/slider';

export function Example() {
  const [value, setValue] = useState([40]);

  return (
    <Slider
      value={value}
      onValueChange={setValue}
      aria-label="Volume"
    />
  );
}
```

### Range (two thumbs)

Pass two values in the array — one `Thumb` is rendered per entry.

```tsx
const [range, setRange] = useState([25, 75]);

<Slider value={range} onValueChange={setRange} aria-label="Price range" />
```

## Sizes

Three track/thumb scales via the `size` prop — `sm`, `md` (default), and `lg`.

```tsx
<Slider defaultValue={[50]} size="sm" aria-label="Small slider" />
<Slider defaultValue={[50]} size="md" aria-label="Medium slider" />
<Slider defaultValue={[50]} size="lg" aria-label="Large slider" />
```

## Show value label

Set `showValue` to float a small value label above a thumb while it's pressed or focused — useful when the
track alone doesn't communicate a precise number.

```tsx
<Slider value={value} onValueChange={setValue} showValue aria-label="Zoom" />
```

## Eased programmatic changes

When `value` changes from outside (a "jump to" button, a reset, syncing from another control), the range
fill and thumb position now ease over ~150ms instead of snapping, so external `setValue` calls read as a
motion rather than a jump. This transition is disabled under `prefers-reduced-motion`.

## Accessibility

Built on `@radix-ui/react-slider` for correct roles, focus management, and pointer/keyboard interaction.

| Key | Action |
| --- | --- |
| `Arrow Left` / `Arrow Down` | Decrease the focused thumb by one step. |
| `Arrow Right` / `Arrow Up` | Increase the focused thumb by one step. |
| `Home` | Move the focused thumb to the minimum value. |
| `End` | Move the focused thumb to the maximum value. |
| `Page Down` | Decrease the focused thumb by a larger step. |
| `Page Up` | Increase the focused thumb by a larger step. |
| `Tab` / `Shift + Tab` | Move focus to the next / previous focusable element. |

Each thumb exposes `role="slider"` with `aria-valuemin`, `aria-valuemax`, and `aria-valuenow`. Thumb
hover/active scale animations, the eased range/thumb transition on programmatic value changes, and the
`showValue` label's spring pop all drop to instant/disabled when `prefers-reduced-motion` is set. The
floating value label is `aria-hidden` — the value is already exposed via `aria-valuenow` on the thumb.
