## Installation

Copy the component from `packages/ui/src/components/foundation/checkbox/` into your project.

```bash
bun add @radix-ui/react-checkbox motion clsx tailwind-merge
```

## Usage

```tsx
import { useState } from 'react';
import { Checkbox } from '@/components/foundation/checkbox';

export function Example() {
  const [checked, setChecked] = useState(true);

  return <Checkbox isChecked={checked} onChange={setChecked} label="Remember me" />;
}
```

Indeterminate ("select all") pattern:

```tsx
const allSelected = selected.length === items.length;
const someSelected = selected.length > 0 && !allSelected;

<Checkbox
  isChecked={allSelected}
  isIndeterminate={someSelected}
  onChange={(checked) => setSelected(checked ? items : [])}
  label="Select all"
/>;
```

Checkbox is fully controlled — there is no internal checked state, so both `isChecked` and `onChange` are required even when the checkbox is disabled.

## Accessibility

Checkbox renders Radix's `<button type="button" role="checkbox">`, which sets `aria-checked` to `true`, `false`, or `"mixed"` (when `isIndeterminate`) automatically — no custom ARIA wiring is needed. Space toggles the box via native button semantics. When `label` is passed, it renders as a `<label>` tied to the box through a `useId()`-generated (or caller-supplied) `id`/`htmlFor` pair; without a `label`, the box falls back to `aria-label="Checkbox"` unless a custom `aria-label` is passed. Clicking a checkbox while `isIndeterminate` is true always resolves to checked, matching native `<input type="checkbox">` behavior. The checkmark/dash path animation and its entrance/exit both drop to a duration of `0` when `prefers-reduced-motion` is set.
