## Installation

Copy the component from `packages/ui/src/components/foundation/textarea/` into your project.

```bash
bun add clsx tailwind-merge
```

## Usage

```tsx
import { Textarea } from '@/components/foundation/textarea';

export function Example() {
  return (
    <Textarea
      label="Bio"
      placeholder="Tell us about yourself"
      helperText="Keep it under 280 characters."
      rows={4}
      isRequired
    />
  );
}
```

## Variants

- **Default** — a bordered surface at rest; focus swaps the border to the accent color and adds a soft, low-opacity glow.
- **Filled** — a quiet `bg-muted` fill with no visible border until focus, when it lifts to `bg-background` with the same accent border + soft glow.
- **Frame** — a gradient-rimmed border (brand → brand-secondary) with a brand-tinted focus glow. Pairs with `Input` and `Button` `variant="frame"` so related form fields read as one family.

## Accessibility

The textarea is a native `<textarea>` element with associated label and description wiring.

| Attribute / element | Purpose |
| ------------------- | ------- |
| `<label htmlFor>` | Associates the visible label with the textarea for screen readers and click-to-focus. |
| `aria-invalid` | Set when `errorText` is present so assistive tech announces the invalid state. |
| `aria-describedby` | Points to helper or error text below the field. |
| `role="alert"` on error | Error message is announced when it appears. |

Keyboard users tab to the field and type normally. Focus is indicated by the container’s accent border and soft glow (same pattern as Input). Disabled fields are skipped in the tab order.
