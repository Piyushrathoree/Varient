## Installation

Copy the component from `packages/ui/src/components/foundation/breadcrumb/` into your project.

```bash
bun add clsx tailwind-merge
```

## Usage

```tsx
import { Breadcrumb } from '@/components/foundation/breadcrumb';

export function Example() {
  return (
    <Breadcrumb>
      <Breadcrumb.List>
        <Breadcrumb.Item>
          <Breadcrumb.Link href="/">Home</Breadcrumb.Link>
        </Breadcrumb.Item>
        <Breadcrumb.Separator />
        <Breadcrumb.Item>
          <Breadcrumb.Link href="/components">Components</Breadcrumb.Link>
        </Breadcrumb.Item>
        <Breadcrumb.Separator />
        <Breadcrumb.Item>
          <Breadcrumb.Page>Breadcrumb</Breadcrumb.Page>
        </Breadcrumb.Item>
      </Breadcrumb.List>
    </Breadcrumb>
  );
}
```

Use `Breadcrumb.Ellipsis` to collapse long trails, and pass children to `Breadcrumb.Separator` for a custom divider.

```tsx
<Breadcrumb.Separator>/</Breadcrumb.Separator>
```

For data-driven trails that auto-collapse past a length, use `Breadcrumb.Auto` — pass `items` and it
renders the first and last few, folding the middle behind a clickable ellipsis that expands the full
trail in place.

```tsx
<Breadcrumb.Auto
  items={[
    { label: 'Home', href: '/' },
    { label: 'Workspace', href: '/workspace' },
    { label: 'Projects', href: '/workspace/projects' },
    { label: 'Varient', href: '/workspace/projects/varient' },
    { label: 'Breadcrumb', isCurrent: true },
  ]}
  maxItems={4}
/>
```

## Accessibility

- Root renders `<nav aria-label="breadcrumb">` so screen readers announce it as a navigation landmark.
- `Breadcrumb.Page` sets `aria-current="page"` on the current segment.
- `Breadcrumb.Separator` uses `role="presentation"` and `aria-hidden="true"` so dividers are skipped by assistive tech.
- `Breadcrumb.Ellipsis` includes visually hidden "More" text for screen readers.
- Links receive a visible focus ring via `focus-visible:ring-ring`.
- `Breadcrumb.Auto`'s collapse toggle is a real `<button aria-label="Show all breadcrumb items">` —
  keyboard-reachable and focus-visible; expanding and item-count changes fade/reflow smoothly,
  instantly under `prefers-reduced-motion`.
